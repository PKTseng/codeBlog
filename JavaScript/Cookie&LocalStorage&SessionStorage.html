# Cookie、LocalStorage、SessionStorage 三者差別

# Cookie

特性：

  • 儲存空間 ４KB
  • 每次都會被帶入 Reaquest Headers 裡面，所以當使用者發送請求時都會被帶到 server 端。
  • 若資料是由 server 產生，那可以設定失效時間。
  • 如果 Cookie 是瀏覽器產生的，預設失效時間就是關閉瀏覽器的時候。

# LocalStorage & SessionStorage

特性＆相同點：

  1. 儲存空間 5Mb 。
  2. 當使用者發送請求時並不會被帶到 server 端。

生命週期：

  1. localStorage：資料在同源瀏覽器下資料永久保存，除非使用者手動清除。
  2. sessionStorage 資料在同源瀏覽器下資料永久保存，在同個分頁下進入到另一個頁面資料仍存在，直到使用者關閉瀏覽器分頁資料才會消失。

# LocalStorage

使用方法：

// setItem 寫入
window.localStorage.setItem('自定義 key 值', '先轉字串再放入要存的 value')


// getIten 拿取
window.localStorage.getItem('自定義 key 值', JSON.stringify(userData));


// removeItem 刪除
window.localStorage.removeItem('自定義 key 值')

存到瀏覽器後我們就可以用 getItem 的方式拿取瀏覽器中 LocalStorage 的資料，但用 getItem 拿到的還是字串型別，所以要用 parase 轉成物件型別才可使用。

let browser_userData = window.localStorage.getItem('userData', userData);

console.log('尚未使用 JSON.parse 轉換=>', browser_userData);
console.log('尚未使用 JSON.parse 轉換的型別=>', typeof browser_userData);

let change_userData = JSON.parse(browser_userData);

console.log('使用 JSON.parse 轉換=>', change_userData);
console.log('使用 JSON.parse 轉換的型別=>', typeof change_userData);

如果需求改成使用者不要儲存登入資料那就用 removeItem 屬性將資料刪除，此需求為指定刪除，並非全部刪除。

let remove_userData = window.localStorage.removeItem('userData');
console.log('刪除使用者資料！', remove_userData);
