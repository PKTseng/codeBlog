# Vue 3 監視(watch) value 的問題

當監視 ref 定義的變數的時候，到底該不加 value ，為釐清這些問題，就實驗了一下。

先講結論，監視 ref 的變數不需要加 value 。

以下來看看兩者差別。

# 監視（watch）變數

先看用 ref 命名的變數，只有純值。

# 1. 不加 value 的

<h2>{{ total }}</h2>
<button @click="total++">+1</button>
1
2
import { ref, reactive, watch } from 'vue';
export default {
  name: 'watchValue',
  setup() {
    let total = ref(10);
    let object = reactive({
      name: 'ken',
      age: 18,
      job: {
        salary: {
          total: 10,
        },
      },
    });

    watch(total, (newValue, oldValue) => {
      console.log(newValue, oldValue);
    });

    return { total, object };
  },
};

# 2. 加 value 的

如果加了 value 就等於在 watch 該屬性的 value 值。

舉例一下，以 JavaScrip 變數的概念來說：

// 給個變數
let name = 'ken'

// 不加 value 的話等於在 watch name 這個屬性。
// 加 value 就是在 watch  'ken' ，這個屬性的 value 值。

# 監視（watch）物件

剛才講的都是監視純值，現在換成監視物件

# 1. 不加 value

import { ref, watch } from 'vue';
export default {
  name: 'watchValue',
  setup() {
    let total = ref(10);
    let object = ref({
      name: 'ken',
      age: 18,
      job: {
        salary: {
          total: 10,
        },
      },
    });

    watch(object, (newValue, oldValue) => {
      console.log(newValue, oldValue);
    });

    return { total, object };

};

# 2. 加 value

這樣就代表 ref 屬性會調用 reactive 屬性，這個 value 會指向 proxy 這個屬性，等於說我使用 .value 的時候就是在用 reactive 屬性。

// 程式碼一樣，只更改 watch 的第一個傳遞參數
// 將 object 改成 object.value

watch(object.value, (newValue, oldValue) => {
  console.log(newValue, oldValue);
});

# 3. deep 屬性

想要深度監視的話也可以使用 deep 屬性，方法也較為簡單。

只要在 watch 屬性加入第三個參數就好。

 watch(
  object.value,
  (newValue, oldValue) => {
    console.log(newValue, oldValue);
  },
  { deep: true }
);
