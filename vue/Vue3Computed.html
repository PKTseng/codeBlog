# Vue3 Computed 計算屬性

在 Vue2 裡面如果有計算屬性的話都會使用 computed ,但在 Vue3 的話寫法跟 Vue2 會有點不太一樣。

# Vue2 的 Computed 寫法

firstName：<input type="text" v-model="person.firstName" />
lastName：<input type="text" v-model="person.lastName" />
fullName：<input type="text" v-model="fullName" />

3
import { reactive } from 'vue';
export default {
  name: 'app',

  computed: {
    fullName() {
      return this.person.firstName + ' - ' + this.person.lastName;
    },
  },

  setup() {
    let person = reactive({
      firstName: 'Tseng',
      lastName: 'ken',
    });

    return {
      person,
    };
  },
};

用Vue devtools (opens new window) 會看到多了一個 computed 計算屬性。在輸入匡內更改，值也會跟著變動。但在 Vue3 用 Vue2 的寫法在日後會很難維護同時也會留下技術債，以下來看看 Vue3 的寫法。

# Vue3 的 Computed 寫法

template 模板內容不變。

在 setup 裡面新增一個 fullName 函數，並將函數返回出來（以下是箭頭函式簡寫）。

import { computed, reactive } from 'vue';
export default {
  name: 'app',

  // computed: {
  //   fullName() {
  //     return this.person.firstName + ' - ' + this.person.lastName;
  //   },
  // },

  setup() {
    let person = reactive({
      firstName: 'Tseng',
      lastName: 'ken',
    });

    let fullName = computed(() => person.firstName + ' - ' + person.lastName);

    return {
      person,
      fullName,
    };
  },
};

但這寫法並不能達到雙向綁定，如下圖

如果我在 fullName 輸入匡內輸入值 firstName 跟 lastName 的值是不會被改變的，必須按照官方文件上 (opens new window)的範例來寫，如下。

import { computed, reactive } from 'vue';
export default {
  name: 'app',
  setup() {
    let person = reactive({
      firstName: 'Tseng',
      lastName: 'ken',
    });

    // let fullName = computed(()=>person.firstName + ' - ' + person.lastName)

    let fullName = computed({
      get() {
        return person.firstName + ' - ' + person.lastName;
      },
      set(value) {
        // 此為陣列解構寫法，如果看不習慣可以用下面註解的方法
        [person.firstName, person.lastName] = value.split(' - ');

        
        // let name = value.split('-');
        // person.firstName = name[0];
        // person.lastName = name[1];
      },
    });

    return {
      person,
      fullName,
    };
  },
};

這樣就可以達到雙向綁定了，但是 fullName 是用 person 透過計算屬性寫出來的，這樣寫會有點怪，乾脆把 fullName 拉到 person 裡面去，如下。

因為 fullName 被更改到 person 底下了所以 template 的 fullName 也要跟著更改。

firstName：<input type="text" v-model="person.firstName" />
lastName：<input type="text" v-model="person.lastName" />
fullName：<input type="text" v-model="person.fullName" />

2
3
import { computed, reactive } from 'vue';
export default {
  name: 'app',
  setup() {
    let person = reactive({
      firstName: 'Tseng',
      lastName: 'ken',
    });

    person.fullName = computed({
      get() {
        return person.firstName + ' - ' + person.lastName;
      },
      set(value) {
        [person.firstName, person.lastName] = value.split(' - ');
      },
    });

    return {
      person
    };
  },
};
