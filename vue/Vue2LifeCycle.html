# beforeCreate & created

beforeCreate： Vue 實體跟 DOM 元素都還沒生成，data、methods、computed 跟 watch 這些都還不能使用。

created： Vue 實體已經生成了但是 DOM 元素還沒生成。

beforeCreate ＆ created 共同點：DOM 元素尚未生成。

以下示範。

<div id="app">
  <h1>{{b}}</h1>
</div>

const vm = new Vue({
  el:'#app',
  data:{
    b:10
  },
  
  beforeCreate() {
    console.log('生命週期鉤子 ＝> beforeCreate');
    console.log(`this.b: ${this.b}`);
    console.log(`this.$el: ${this.$el}`);
  },
  
  created() {
    console.log('生命週期鉤子 ＝> created');
    console.log(`this.b: ${this.b}`);
    console.log(`this.$el: ${this.$el}`);
  },
  
})

# beforeMount & mounted

beforeMount：Vue 實體還沒掛載到 DOM 元素上。 mounted：Vue 實體已掛載到 DOM 元素上。

beforeMount 跟 mounted 共同點：Vue 實體已生成。

以下示範。

const vm = new Vue({
  el:'#app',
  data:{
    b:10
  },
  
  beforeMount() {
    console.log('生命週期鉤子 ＝> beforeMount');
    console.log(`beforeMount 的 this.b: ${this.b}`);
    console.log(this.$el.outerHTML);
  },
  
  mounted() {
    console.log('生命週期鉤子 ＝> mounted');
    console.log(`mounted 的 this.b: ${this.b}`);
    console.log(this.$el.outerHTML);
  },
})

# beforeUpdate & updated

我們多設一個互動按鈕來更新數據並看看 beforeUpdate & updated 的差別。

beforeUpdate 是 Vue 實體資料已經更新但尚未更新到 DOM 元素上，如下圖紅匡處，到了 updated 後 Vue 實體資料會更新到 DOM 元素上。

以下示範。

<div id="app">
  <h1>{{b}}</h1>
  <button @click='b++'>b+1</button>
</div>

  el:'#app',

    console.log(`this.b: ${this.b}`);
    console.log(`this.$el: ${this.$el}`);

  },
  
  updated() {
    console.log('生命週期鉤子 ＝> updated');
    console.log(`this.b: ${this.b}`);
    console.log(`this.$el: ${this.$el}`);

# beforeDestroy & destroyed

beforeDestroy 意思是要把 Vue 實體即將銷毀，如果有資料要銷毀的話可以放在這裡。

如果有在 mounted 使用 addEventListener 那 beforeDestroy 就可以用 removeEventListener 來銷毀。

destroyed 是 Vue 實體資料已經銷毀。

以下示範。

<div id="app">

  <h1>{{b}}</h1>
  
  <button v-on:click="b++">b++</button>
  
  <button v-on:click="$destroy()">Destroy instance</button>
</div>
